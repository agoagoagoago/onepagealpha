## What this is
**OnePage Alpha** — a boutique, editorial financial-research site that publishes free **visual annual-report infographics** ("visual briefs") for busy investors. Positioning: *"Visual annual report intelligence for busy investors."* Educational only — **not financial advice**.

Business model / goals:
- Give away company infographics (PNG/JPEG preview + downloadable PDF).
- Capture emails via an **email-gated download** before sending the link.
- Accept voluntary support via **Buy Me a Coffee**.
- Let visitors **request a company** and **contact** for general enquiries.
- Track engagement with **Fathom Analytics**.

## Tech stack
- **Next.js 14** (App Router) · **TypeScript** · **Tailwind CSS v3**
- **Resend** for all transactional email (server-side API routes)
- **Fathom Analytics** (privacy-friendly, script via `next/script`)
- Deployed on **Vercel**; repo `github.com/agoagoagoago/onepagealpha`; domain `onepagealpha.com`
- No database / auth / CMS. All content lives in `data/companies.ts`.

## Routes (App Router)
- `/` — homepage: hero (text only, no CTA buttons) → featured company (single **View Full Brief** CTA) → library teaser → Request CTA → disclaimer → footer
- `/companies` — library: client-side tag filter (`CompanyExplorer`) + card grid + Request CTA
- `/companies/[slug]` — individual brief (SSG via `generateStaticParams`): metadata header, `reportTitle` headline, summary, `InfographicPreview` (image click opens the download gate), Download / Buy Me a Coffee / Back buttons, `CoverageGrid`, related briefs, disclaimer, per-company JSON-LD
- `/request` — "Request a Company" form → `POST /api/request-company`
- `/contact` — general contact form → `POST /api/contact`
- `/api/download-request` — gated download email (Resend)
- `/api/request-company` — company request emails (Resend)
- `/api/contact` — contact emails (Resend)
- `/sitemap.xml`, `/robots.txt` — generated by `app/sitemap.ts` / `app/robots.ts`
- `app/icon.png` (favicon) + `app/apple-icon.png` — auto-detected square icons cropped from the logo mark

### How to add / update a company
1. Add image → `public/infographics/<slug>-<date>.jpg` and PDF → `public/downloads/<slug>-<date>.pdf`.
2. **Compress the preview image** to full-res JPEG q88 (~0.4–0.9 MB). Convention used so far: PowerShell + `System.Drawing` (load PNG, save with JPEG encoder quality 88). Originals are ~2–5 MB; never ship those.
3. Add/edit one object in `data/companies.ts` (set `infographicImage` to the `.jpg`, `downloadFile` to the `.pdf`). Everything else (pages, library, sitemap, related, SEO) updates automatically.
4. Featured = the company with `isFeatured: true`, else most recent by `reportDate`. Only one should be featured.
5. Missing files degrade gracefully (placeholder card) — never crash.

## Email (Resend) — Formspree fully removed
All three forms POST JSON to a Next.js API route that validates server-side, rate-limits, and sends via Resend.
- **Download gate** (`GatedDownload` → `/api/download-request`): captures email, emails the user the **download link + a Buy Me a Coffee link**, optional owner notification. Download is **email-only** — there is no in-modal download button (removed by request). Reveals a success/confirmation message.
- **Request a Company** (`/api/request-company`): owner notification + requester confirmation. Name + Email + Requested Company Name are **required**.
- **Contact** (`/api/contact`): owner notification + sender confirmation. Email + enquiry type + subject + message required.

Server conventions in API routes:
- `isValidEmail()` (trim, ≤254, no spaces, regex `^[^\s@]+@[^\s@]+\.[^\s@]{2,}$`), normalize to lowercase.
- Best-effort **in-memory rate limit**: 5/IP and 3/email per 10 min → 429. (Per-instance only; for scale use Upstash Redis / Vercel KV.)
- If `RESEND_API_KEY`/`RESEND_OWNER_EMAIL` missing → safe 5xx message, never leak keys/stack traces.
- Confirmation email failure is logged only, never fails the request.

## Environment variables
| Var | Scope | Notes |
|---|---|---|
| `RESEND_API_KEY` | **server-only** | NEVER prefix with `NEXT_PUBLIC_`; never use in client code |
| `RESEND_FROM_EMAIL` | server | Use verified domain in prod (e.g. `OnePage Alpha <hello@onepagealpha.com>`); falls back to `onboarding@resend.dev` |
| `RESEND_OWNER_EMAIL` | server | Required for request + contact; download notification recipient. Currently `lookforadrian@gmail.com` |
| `NEXT_PUBLIC_SITE_URL` | public | Builds absolute download/sitemap URLs (`https://onepagealpha.com`) |
| `NEXT_PUBLIC_FATHOM_SITE_ID` | public | Defaults to `RAWISURF` in code; override for staging |
| `NEXT_PUBLIC_FATHOM_SCRIPT_URL` | public | Defaults to `https://cdn.usefathom.com/script.js` |

`.env.example` is the source of truth for what to set. Set these in Vercel → Settings → Environment Variables, then redeploy.

## Fathom tracking (`lib/fathom.ts`)
Always `window.fathom.trackEvent` — **never `trackGoal`**; fails silently if Fathom absent; dev logs only. Events fire a general name + a company-specific `_<slug>` variant where relevant (slug sanitized via `toSafeEventName`).
- Downloads: `infographic_download_clicked`, `download_<slug>`, `download_gate_opened`, `download_email_submitted`, `download_email_invalid`, `download_email_failed` (each with `_<slug>`). Note: `gated_download_completed` helper exists but is no longer called (no in-modal download button).
- Support: `buy_me_a_coffee_clicked`, `coffee_<slug>`
- Request: `request_company_clicked` (+ `request_company_homepage/library/company_page/footer`, `request_company_from_<slug>`), `request_company_form_started/submitted/failed/invalid`
- Contact: `contact_form_started/invalid/submitted/failed`
- Email links (in Resend emails) are NOT Fathom-tracked (Fathom is browser-only).

## Design system
- Colors (Tailwind, `tailwind.config.ts`): `ivory #FBFAF7` (bg), `ink #1A2433` (navy text), `ink-soft`/`ink-muted`, `gold #B08D3C` (accent), `line #E7E3DA` (borders), `paper #FFFFFF`.
- Fonts (`next/font`): **Fraunces** serif (`font-serif`, headings) + **Inter** (`font-sans`, body). Modal/dialog titles intentionally use **sans** (Fraunces' curly glyphs looked off at small UI sizes).
- Premium, calm, editorial. Subtle borders, generous whitespace, subtle hover only. Logo is a wide ~3:1 lockup (transparent PNG in `public/brand/`).
- Images: `InfographicPreview`/cards use a plain `<img>` via `ImageWithFallback` (graceful `onError` placeholder, `loading="lazy"`, `decoding="async"`) — deliberately **not** `next/image`, so a missing file can't break layout. `BrandLogo` does use `next/image`.

## SEO
- Title template `%s | OnePage Alpha` (homepage has a descriptive default; `/contact` uses `title.absolute`). Per-page descriptions, canonical, OG/Twitter (company OG = its infographic).
- `app/sitemap.ts` (all routes + company slugs) and `app/robots.ts` (allow all, disallow `/api/`, points to sitemap).
- JSON-LD: Organization + WebSite site-wide; BreadcrumbList + CreativeWork per company page.

## Compliance (always)
Educational, public-disclosure framing. NEVER: buy/sell/hold recs, price targets, "undervalued/overvalued", performance/guarantee claims, personalized advice, testimonials, fabricated numbers. Disclaimer component appears on every page + in emails.

## Working conventions / gotchas
- **Environment is Windows + PowerShell.** `git push` prints progress to stderr (shows red) but succeeds — check the `… -> main` line, not the color.
- Committing with **double quotes** in a PowerShell here-string breaks parsing — use a temp message file (`git commit -F`) or avoid quotes in the message.
- Workflow each task: edit → `npm run build` (must pass clean: no TS/lint errors) → `git add` → commit → `git push origin main`. Vercel auto-deploys `main`.
- Fathom custom events generally don't fire on `localhost` — verify on the deployed HTTPS site.
- Line-ending warnings (LF→CRLF) on `git add` are normal noise.
